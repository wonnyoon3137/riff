## F1. 공연 목록 탐색

### F1.1 공연 카드 목록 표시
- 사용자가 메인 페이지에 진입하면 공연 목록을 카드 그리드 형태로 표시한다.
- **첫 진입 시 기본 노출**: 오늘부터 30일 이내 + 전국 + 전체 장르
- **카드 표시 항목** (KOPIS `pblprfr` 응답 기준)
  - 포스터 이미지 (`poster`)
  - 공연명 (`prfnm`)
  - 공연 기간 (`prfpdfrom` ~ `prfpdto`)
  - 공연장명 (`fcltynm`)
  - 지역 (`area`)
  - 장르 (`genrenm`)
  - 공연 상태 뱃지 (`prfstate`: 공연중/공연예정/공연완료)

### F1.2 페이지네이션 — 무한 스크롤
- 스크롤이 하단에 가까워지면 (Intersection Observer) 다음 페이지 자동 로드
- 페이지당 30개 (KOPIS는 최대 100개까지 가능)
- **상태 복원 요구사항 (F1.2 ↔ F3.3 연계)**
  - 상세 페이지 진입 후 뒤로가기 시 스크롤 위치 및 로드된 페이지 수 복원
  - 구현 방안은 화면설계서 / 데이터 모델 단계에서 확정 (예: sessionStorage, React Query 캐시 활용)
- **URL 동기화**
  - 필터 상태는 쿼리스트링에 반영 (공유 가능)
  - 페이지 번호는 URL에 미포함 (무한 스크롤 특성상). 단, 직접 진입 시 deep link로 특정 페이지 복원은 미지원.

### F1.3 로딩 / 빈 상태 / 에러 처리
- **로딩**
  - 첫 로드: 스켈레톤 카드 그리드
  - 추가 로드 (무한 스크롤): 하단에 스피너
- **빈 상태**: "조건에 맞는 공연이 없습니다" 메시지 + 필터 초기화 버튼
- **에러**: "공연 정보를 불러오지 못했습니다" 메시지 + 재시도 버튼

## F2. 필터링

### F2.1 월별 필터
- 이번 달 / 다음 달 / 그 다음 달 / 직접 선택 (날짜 범위 피커)
- KOPIS `stdate`, `eddate` 파라미터에 매핑
- **31일 초과 시**: UI에서 31일 이내로 제한 (v0.1)
  - 사용자가 31일을 초과해 선택하면 토스트로 "최대 31일까지 선택 가능합니다" 안내
  - v0.2에서 분할 호출 방식으로 확장 예정

### F2.2 지역 필터 — 다중 선택
- 시도 단위 1단계 → 구군 단위 2단계 (드릴다운)
- KOPIS `signgucode` (시도), `signgucodesub` (구군) 매핑
- **다중 선택 허용** — 시도 또는 구군을 여러 개 선택 가능
- KOPIS API는 단일 지역 코드만 받으므로, 다중 선택 시 클라이언트에서 병합 처리
  - 구현 방안: 선택된 지역별로 병렬 API 호출 후 결과 병합 (BFF 레이어에서 처리)
  - 데이터 모델 / API 명세 단계에서 상세화
- "전국" 선택 시 다른 지역 선택 해제

### F2.3 장르 필터
- KOPIS `shcate` 코드 매핑 (예: AAAA=연극, BBBB=뮤지컬, CCCA=대중음악 등)
- 칩(chip) 형태로 다중 선택 허용
- 장르 코드 한글 매핑은 별도 문서(`docs/api/kopis-codes.md`)에서 관리

### F2.4 공연장 필터 — 사전 동기화
- 공연장명 자동완성 입력
- 선택 시 KOPIS `prfplccd` 파라미터로 전달
- **공연장 마스터 데이터**: KOPIS `prfplc` (공연시설 목록 조회 서비스)에서 사전 동기화하여 자체 DB에 저장
  - 동기화 방식: 초기 일괄 fetch + 주기적 갱신 (스케줄 빈도는 데이터 모델 단계에서 결정)
  - 이 자체 DB가 v0.2 자체 인덱스 구축의 기반이 됨

### F2.5 필터 조합 동작
- **필터 변경 즉시 반영** (debounce 300ms)
- 모든 필터는 URL 쿼리스트링에 반영 (공유/북마크 가능)
- "필터 초기화" 버튼 제공

## F3. 공연 상세 페이지

### F3.1 상세 정보 표시
- 진입 경로: 카드 클릭 → `/performances/[mt20id]`
- KOPIS `pblprfr/{mt20id}` API 호출
- **표시 항목**
  - 포스터 (대형)
  - 공연명, 공연 기간, 장르, 관람 연령
  - 공연장 + 주소 (지도는 v0.1 제외)
  - 출연진 (`prfcast`) — 텍스트 그대로
  - 줄거리 (`sty`)
  - 티켓 가격 (`pcseguidance`) — 텍스트 그대로
  - 공연 시간 (`dtguidance`)
  - 소개 이미지 갤러리 (`styurls`)
  - 기획/제작/주최/주관 정보

### F3.2 예매처 링크
- KOPIS `relates` 배열 순회하여 외부 링크 버튼으로 노출
- 예: "대학로티켓닷컴에서 예매", "티켓링크에서 예매"
- `target="_blank"` + `rel="noopener noreferrer"`
- 예매처가 없는 경우 "예매 정보 없음" 메시지

### F3.3 목록으로 돌아가기
- 뒤로가기 시 **이전 필터 / 스크롤 위치 / 로드된 페이지 수 보존**
- 구현 방안은 화면설계서 / 데이터 모델 단계에서 확정

## F4. 정렬

- **정렬 옵션**
  - 시작일 가까운 순 (디폴트)

## 사용자 시나리오

### 시나리오 1: 다음 달 서울 + 경기 뮤지컬 찾기
1. 메인 진입 → 30일 이내 전국 공연이 디폴트로 보임
2. 월별 필터 "다음 달" 선택 → 자동 갱신
3. 지역 필터 "서울" + "경기" 다중 선택 → 자동 갱신
4. 장르 필터 "뮤지컬" 칩 클릭 → 자동 갱신
5. 마음에 드는 공연 카드 클릭 → 상세 페이지
6. "인터파크티켓에서 예매" 버튼 → 외부 사이트 이동
7. 뒤로가기 → 이전 스크롤 위치 + 필터 그대로 복원

### 시나리오 2: 특정 공연장의 공연 보기
1. 메인 진입
2. 공연장 필터에 "롤링홀" 입력 → 자동완성에서 선택
3. 자동 갱신된 결과 확인

## 비기능 요구사항 (NFR)

- **반응형**: 모바일 / 태블릿 / 데스크톱 지원
- **성능**: 첫 페이지 로드 3초 이내 (KOPIS API 응답 기준)
- **접근성**: 키보드 네비게이션, alt 텍스트, 의미 있는 헤딩 구조
- **SEO**: 공연 상세 페이지는 메타 태그 동적 생성 (v0.1 우선순위 낮음)
