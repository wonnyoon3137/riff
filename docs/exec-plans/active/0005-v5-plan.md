### DEC-P4 planner 추천 근거

- **NextAuth.js(Auth.js)는 Next.js App Router와 네이티브 통합**. 카카오/구글 OAuth provider가 내장. 세션 관리, CSRF, callback URL 등을 프레임워크가 처리.
- JWT stateless(A)는 로그아웃/토큰 무효화가 어렵고, 서버 세션(B)은 직접 구현 부담이 큼. Auth.js는 JWT/DB 세션 모두 지원하되 기본 설정으로 빠르게 시작 가능.
- Riff의 현재 스택(Next.js App Router + BFF)에 가장 자연스럽게 맞는다.

### DEC-P4b planner 추천 근거

- 현재 Venue/Artist가 SQLite 파일 DB(better-sqlite3)로 운영 중. **동일 패턴 재사용**으로 설계 시간 절약 + 일관성.
- 사용자 수가 대규모가 아닌 MVP 단계에서 SQLite는 충분. PostgreSQL 등 외부 DB는 인프라 비용/운영 부담을 추가하지만 현 단계에서는 과설계.
- Auth.js의 SQLite adapter(drizzle/prisma) 또는 커스텀 adapter로 연결 가능.

> DEC-P4, DEC-P4b는 planner 추천이다. 사람이 다른 선택지를 원하면 변경.

**스택 결정:**
- **Auth.js (NextAuth.js v5)**: Next.js App Router 네이티브. 카카오/구글 provider 내장.

**비저장 원칙 완화 범위:**
- User/Account/Session은 SQLite에 영속 저장 (Venue/Artist 선례 확장).
- 공연 데이터는 여전히 비저장(KOPIS 실시간 조회). Follow/Record는 v5 밖(P5/P6).

**제약:**
- 카카오 OAuth: [Kakao Developers](https://developers.kakao.com) 앱 등록 필요. Client ID/Secret 환경 변수.
- 구글 OAuth: [Google Cloud Console](https://console.cloud.google.com) OAuth 2.0 Client 등록 필요.
- 약관/개인정보처리방침: 최소한의 텍스트 작성 필요 (MVP = 간단한 동의 체크).
