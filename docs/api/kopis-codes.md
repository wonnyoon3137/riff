## 1. 공연 상태 — `prfstate`  ✅ 확인됨

요청 시 **코드**(01/02/03), 응답 `prfstate` 필드는 **라벨 문자열**(예 "공연중")로 반환됨.
출처: 공통코드 PDF(공연상태 코드) + 공연목록 응답 예시.

| 코드 | 라벨(응답) | 내부 enum | UI 뱃지 색(M3 role) |
|---|---|---|---|
| `01` | 공연예정 | `UPCOMING` | `primary-container` |
| `02` | 공연중 | `ONGOING` | `secondary-container` |
| `03` | 공연완료 | `ENDED` | `surface-variant`(흐리게) |

> 정규화 주의: 목록/상세 응답은 `prfstate`를 **라벨**("공연중")로 주므로, 라벨→enum 매핑을 사용한다. 요청 필터로 보낼 때만 코드(02 등)를 쓴다. v0.1은 상태 필터를 노출하지 않으므로(전체), 응답 라벨 매핑만 필요.

## 2. 장르 — `shcate`(요청 코드) / `genrenm`(응답 라벨)  ✅ 확인됨

공연목록·상세에서 사용. 요청은 `shcate` 4자리 코드, 응답은 `genrenm` 한글 라벨.
출처: 공통코드 PDF(장르 코드, 변수명 shcate / 필드명 genrenm).

| UI 라벨(genrenm) | 내부 enum | `shcate` 코드 | 검증 |
|---|---|---|---|
| 연극 | `THEATER` | `AAAA` | ✅ |
| 무용(서양/한국무용) | `DANCE` | `BBBC` | ✅ |
| 대중무용 | `POPULAR_DANCE` | `BBBE` | ✅ |
| 서양음악(클래식) | `CLASSIC` | `CCCA` | ✅ |
| 한국음악(국악) | `KOREAN_MUSIC` | `CCCC` | ✅ |
| 대중음악 | `POPULAR_MUSIC` | `CCCD` | ✅ |
| 복합 | `COMPLEX` | `EEEA` | ✅ |
| 서커스/마술 | `CIRCUS_MAGIC` | `EEEB` | ✅ |
| 뮤지컬 | `MUSICAL` | `GGGA` | ✅ |

> 정정: shcate에 "기타"는 없다(이전 `ETC` enum 삭제). 위 9개가 공식 전체 목록.
>
> ⚠️ 주의 — **예매상황판/통계용 장르 코드(`catecode`)는 별도**다. 같은 PDF에서 catecode는 대중무용을 `BBBR`로 쓰고 `KID`(아동)·`OPEN`(오픈런)을 추가로 둔다. **Riff(공연목록 pblprfr)는 `shcate`를 쓰므로 위 표만 사용**한다. catecode와 혼동 금지.

### 2.1 v0.1 노출 장르 (∴ 기획서 §4.2)
기획서 명시 5종: **뮤지컬(GGGA)·연극(AAAA)·클래식(CCCA)·무용(BBBC)·대중음악(CCCD)**. 나머지(국악 CCCC / 대중무용 BBBE / 복합 EEEA / 서커스·마술 EEEB) 노출 여부는 제품 결정(권장: 전부 노출해도 무방).

## 3. 시도 코드 — `signgucode`  ✅ 확인됨

**행정표준코드 앞 2자리.** 공연목록(pblprfr) 요청 파라미터 `signgucode`(크기 2). 응답에는 코드가 아니라 `area`(시도명 라벨)로 옴.
출처: 공통코드 PDF — "지역(시도)코드 (행정표준코드 앞 2자리), 변수명 signgucode".

| `30` | 대전광역시 |
| `31` | 울산광역시 |

| `41` | 경기도 |

| `43` | 충청북도 |
| `44` | 충청남도 |
| `45` | 전라북도 |

> 17개 시도 전량 확정. "전국"은 `signgucode` 파라미터를 보내지 않음(∴ D4). 응답 `area`는 "서울특별시"처럼 라벨로 오므로 코드↔라벨 매핑에 본 표 사용.
>
> ⚠️ **별도 주의:** 같은 PDF에 "지역(시도)코드 (예매상황판용) 변수명 `area`" 표가 따로 있는데, 그쪽은 충청을 `43|44`로 묶는 등 **통계용**이라 코드 체계가 다르다. Riff는 위 `signgucode` 표(행정표준 앞 2자리)만 사용한다.

## 4. 구군 코드 — `signgucodesub`  ✅ 확인됨 (체계) / 일부 표만 수록

**행정표준코드 앞 4자리.** 공연목록 요청 파라미터 `signgucodesub`(크기 4). 응답에는 코드 없이 `area`(시도명)만 옴. 공연시설목록은 `gugunnm`(구군명 라벨) 반환.
출처: 공통코드 PDF — "지역(구군)코드 (행정표준코드 앞 4자리), 변수명 signgucodesub".

### 4.2 나머지 시도 구군
공통코드 PDF(p4~p10)에 부산(26xx)·대구(27xx)·인천(28xx)…제주(50xx)까지 전량 수록. **구현 시 PDF에서 전체 구군 표를 추출해 `src/domain/kopis-codes.ts`(또는 데이터 파일)로 일괄 생성**한다. (행정표준코드 법정동 앞 4자리와 동일하므로 행정표준코드 데이터셋으로도 생성 가능.)

> v0.1 결정 권장: 구군 드릴다운을 v0.1에 포함(F2.2)하려면 전체 구군 표가 필요하다. 데이터 양이 많으므로 PDF/행정표준코드에서 생성. 부담되면 시도 단위까지만 v0.1, 구군은 v0.2로 축소하는 옵션도 검토(제품 결정).

## 7. 여전히 확인/주의가 필요한 항목

대부분 확정됐다. 남은 항목은 다음과 같다(추측으로 채우지 않음).

- [ ] **전체 구군(signgucodesub) 표 데이터화** — 서울 외 시도는 PDF(p4~p10)에서 추출 필요. 값 자체는 공식 확정, 단 본 문서에 전량 전사는 생략(데이터 파일로 생성 권장).
- [ ] **`signgucode` 크기 불일치(문서상)** — 공연목록 명세는 `signgucode` 크기 2, 공연시설목록 명세는 크기 4로 표기됨. 실제 값은 둘 다 2자리(11 등). 명세 표기 오타로 보이나, 구현 시 2자리 사용으로 확정(시설목록 호출도 2자리 시도코드 전달).
- [ ] **예매상황판 catecode/area 코드** — Riff는 미사용. 통계 기능을 추후 도입할 때만 확인.

> 확정된 코드는 구현 시 단일 매핑 모듈(`src/domain/kopis-codes.ts`)로 관리한다.
