## 1. v0.6 → v0.7 무엇이 달라졌나

v0.6은 "팔로우/언팔로우 + 팔로잉 목록"이었다. v0.7은 **"팔로잉 아티스트의 다가오는 공연을 한 화면에 모아보는 피드 페이지"** 다.

### 1.1 새로 도입한 것

- **F-2 docs 정비**: features.md F11 Following Feed, screens.md S5 Following Feed 페이지, feature-checklist.md 수정(이전 세션 AI가 잘못 [x]로 표시한 항목 수정).
- **P6-1 BFF API**: `GET /api/follows/feed` — 팔로잉 artist_ids → performance_artists mt20ids → KOPIS 3기간 분할 병렬 호출 → 교차 필터 + 정렬 + 중복 제거. `hasFollowedArtists` 플래그로 빈 상태 2종 구분.
- **P6-2 Follow Feed UI**: `/following-feed` 페이지(S5). 빈 상태 2종(팔로잉 없음 / 공연 없음). AppBar 드롭다운 "팔로잉 공연" 링크.
- **P6-3 E2E**: 5개 시나리오(비로그인 redirect, 공연 카드 렌더, 팔로잉 없음 빈 상태, 공연 없음 빈 상태, AppBar 링크 이동).

## 2. 잘한 것

### 2.1 R17/R18 교훈 전량 반영

| 교훈 | v7 반영 | 준수 |
|---|---|---|
| **R17** | pnpm 간접 의존성 직접 import 금지 | **이행** (next-auth/jwt 등 직접 의존성만 사용) |
| **R18** | 중간 상태 파일 유지 | **이행** (세션 재시작 없이 단일 세션 완료) |

### 2.2 phantom 이슈 조기 발견 및 정리

이전 세션 AI가 존재하지 않는 커밋/브랜치를 참조하는 이슈(#91)를 남겼다. 세션 시작 시 git log와 이슈 대조로 phantom 이슈를 조기 발견, 즉시 close하고 올바른 기반에서 v7을 시작했다.

### 2.3 feature-checklist.md 정확성 복원

이전 세션이 미구현 기능을 [x]로 잘못 표시해 놓았다. F-2 docs 작업에서 checklist를 정확히 수정했다(v0.6 기준으로 되돌림).

### 2.4 hasFollowedArtists 플래그로 빈 상태 2종 구분

팔로잉 없음 vs 공연 없음을 UI에서 구분하기 위해 API 응답에 `hasFollowedArtists: boolean`을 추가했다. UI 클라이언트가 추가 API 호출 없이 정확한 빈 상태 메시지를 표시할 수 있게 됐다.

### 2.5 D3 우회 패턴 재활용

KOPIS 31일 제한(D3)을 3개 기간(30일 × 3 = 90일)으로 분할 병렬 호출하는 방식으로 90일 피드를 구현했다. 기존 `mapWithConcurrency` 유틸을 재사용해 코드 중복 없이 처리했다.

### 2.6 Playwright strict mode — exact: true

E2E에서 `getByRole("heading", { name: "팔로잉 공연" })`이 공연 카드 제목(h3)도 포함 매칭해 strict mode 위반이 발생했다. `{ name: "팔로잉 공연", exact: true }` 로 수정해 h1만 정확히 매칭하도록 했다. 한국어 부분 문자열 매칭 주의 필요.

## 3. 못한 것 / 교훈

### 3.1 Playwright heading strict mode 위반 초기 미예방

- **사실**: 공연 카드 제목에 페이지 제목의 부분 문자열("팔로잉 공연")이 포함되어 strict mode violation 발생. 초기 E2E 작성 시 `exact: true`를 기본으로 쓰지 않았다.
- **교훈(R19)**: 페이지 제목과 컨텐츠 제목이 겹칠 수 있는 경우, `getByRole("heading", { exact: true })` 를 기본 패턴으로 사용한다.
